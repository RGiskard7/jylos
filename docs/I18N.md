# Internationalization

Español: [es/I18N.md](es/I18N.md)

Jylos currently ships with two fully supported UI locales:

- `messages_en.properties` — English
- `messages_es.properties` — Spanish

The language switch remains in **View** rather than Preferences because it is an immediate presentation toggle and users often change it temporarily without treating it as a deeper app setting.

## Add a new language

1. Create a new bundle under `jylos/src/main/resources/com/example/jylos/i18n/`.
   Example: `messages_fr.properties`.
2. Copy all keys from `messages_en.properties`.
3. Translate values only; do not rename keys.
4. Keep placeholders unchanged.
   Examples: `{0}`, `{1}`, `%s`, embedded Markdown examples, shortcut text.
5. Add the new language option to the UI language menu in:
   - `jylos/src/main/resources/com/example/jylos/ui/view/ToolbarView.fxml`
   - `jylos/src/main/java/com/example/jylos/ui/controller/ToolbarController.java`
   - `jylos/src/main/java/com/example/jylos/ui/controller/AppSettings.java`
6. Verify startup locale loading in `jylos/src/main/java/com/example/jylos/Main.java` still resolves the new bundle.
7. Run:
   - `mvn -f jylos/pom.xml -q -DskipTests compile`
   - `mvn -f jylos/pom.xml -q test`
8. Manually review the main surfaces:
   - menus and toolbar
   - editor / preview
   - dialogs and overlays
   - graph, canvas, Git, insights, plugins

## Rules

- English and Spanish must remain complete; no mixed-language regressions.
- New UI strings must be added to all shipped bundles in the same change.
- Fallback text in Java code is acceptable only as a safety net, not as the primary translation source.
- If a new feature introduces many strings, group them by feature in the bundle file to keep maintenance predictable.
